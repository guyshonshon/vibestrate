Amaco rename candidates
Checked: 2026-05-27

Context
Amaco is a local-first supervisor for coding-agent CLIs. It runs the tools a
developer already has, such as Claude Code, Codex, Aider, Ollama, and OpenCode,
through a visible plan -> architect -> implement -> validate -> review -> fix
-> verify loop in an isolated git worktree. The promise is not "another agent
framework." The promise is: real software work, supervised, auditable, local,
reviewed, tested, and never merged without the developer.

Naming direction
The name should say that this is a tool for running and supervising coding
agents. Not just "merge status", not abstract trust language, and not random
brand vocabulary.

The useful naming territories are:
- Conductor: coordinates a crew through a sequence without doing the work itself.
- Supervisor: says the control model directly.
- Workbench: a hands-on developer surface for agent work.
- Runbook: a guided operating procedure, close to Guides/Flows.
- Crew: the clearest bridge to CrewAI while staying honest to the product.
- Worktree: the local-first execution boundary that makes this different.

Final naming call
If I had to rename the project today, I would pick CrewConductor.

Reason:
- It says what the tool does: conducts a crew.
- It is connected to CrewAI-style language without copying CrewAI.
- It implies supervision and sequencing, not just autonomous agents.
- It fits the product's flow: plan -> architect -> implement -> validate ->
review -> fix -> verify.
- It leaves room for Guides, provider routing, worktree execution, dashboards,
and human gates.

LangGraph comparison
Short answer: LangGraph can overlap with the underlying orchestration pattern,
but it does not already replace this product.

LangGraph's official positioning is that it is a low-level orchestration
framework and runtime for long-running, stateful agents. Its core strengths
include durable execution, streaming, human-in-the-loop workflows, memory,
persistence, and deployment/observability through the LangChain/LangSmith
ecosystem.

That means a team could build an Amaco-like workflow on LangGraph. They could
model plan, build, review, fix, and verify as graph nodes. They could add human
interrupts. They could persist state. They could stream events.

But Amaco's product is different:
- Amaco is an end-user developer tool, not a low-level framework.
- Amaco runs existing coding-agent CLIs instead of asking users to rebuild
agents in a Python/JS framework.
- Amaco's default workflow is software-engineering-specific: plan, architect,
implement, validate, review, fix, verify.
- Amaco uses isolated git worktrees as the mutation boundary.
- Amaco treats tests, typechecks, build commands, diffs, approvals, budgets, and
run history as first-class product surfaces.
- Amaco never hosts the user's keys, code, telemetry, or merge decision.
- Amaco is open-source, local-first, and non-profit by design.

So the competitive answer is:
LangGraph is infrastructure for building agent systems.
CrewAI is a framework/platform for building agent teams and flows.
Amaco should be the local-first supervisor for coding agents already sitting on
a developer's machine.

How to compete
1. Own a narrower category.
Do not compete as a general multi-agent framework. The stronger category is:
"local-first coding-agent supervisor" or "the review and verification layer for
AI coding agents."

2. Make the default workflow impossible to miss.
The product should scream: plan -> build -> review -> verify. LangGraph can be
configured to do that, but Amaco should make it the default experience.

3. Make git and validation the moat.
Worktree isolation, validation commands, diff inspection, review findings, phase
rewind, and merge-ready reports are more valuable to developers than generic
agent graph primitives.

4. Stay provider-neutral.
Claude Code, Codex, Aider, Ollama, OpenCode, and future CLIs should be treated
as interchangeable workers. That is a stronger open-source story than locking
users into one model framework.

5. Be the trust layer.
No hosted keys. No telemetry. No payments. No auto-push. No auto-merge. Human
approval gates. Clear cost/token ledger. This is how a non-profit open-source
project can compete against better-funded platforms.

6. Build the best run inspector.
The dashboard should feel closer to a trace viewer and code-review cockpit than
an AI chat UI. The winning surface is the timeline, diff, validation log,
decision ledger, and rework controls.

7. Treat LangGraph and CrewAI as adjacent, not enemies.
It may be useful later to export Amaco run events or provide a LangGraph/CrewAI
integration, but the core brand should remain independent: Amaco's replacement
name should signal supervised code changes, not generic agent orchestration.
