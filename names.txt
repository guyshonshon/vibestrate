Amaco rename candidates
Checked: 2026-05-27

Constraint
- Maximum 7 characters.
- No git/worktree-first names. Worktrees are implementation, not the brand.
- Name should point at agent runs, supervision, orchestration, provider control,
  or the run cockpit.

Availability standard
All names below passed exact checks for the lowercase compact slug:
- npm package: not registered.
- GitHub account/org: not registered.
- GitHub repositories: no exact repository-name hit.
- Homebrew: absent from formula and cask tokens.

This is not trademark, domain, or social-handle clearance. Before launch, reserve
the GitHub org, npm package, Homebrew formula path, domain, and socials.

Best overall: AgntOps
Best technical CLI name: OrchCtl
Best multi-provider name: ProvMux
Best dashboard/cockpit name: AgBoard
Best supervisor name: SupDeck

Final naming call
If the name must be 7 characters or less, I would pick AgntOps.

Reason:
- It is short, but still readable as "agent ops."
- It describes the product category: operations/control for coding agents.
- It does not over-index on worktrees.
- It leaves room for provider routing, approval gates, validation, review, and
  run history.

If you want the more open-source CLI style, pick OrchCtl instead.

Competitive positioning
LangGraph can model similar workflows, but it is infrastructure for building
stateful agent graphs. Amaco should not compete as another graph framework.
Amaco's category should be: local-first supervised coding-agent runs.

AgentCrew is broader: a multi-agent chat/workbench with roles, tools, MCP, web,
automation, and app surfaces. Amaco is narrower and sharper: it supervises real
software changes through plan -> implement -> validate -> review -> fix -> verify,
using the coding-agent CLIs a developer already has.

How to compete:
- Own the software-change workflow, not generic multi-agent chat.
- Make validation, diffs, review findings, approvals, budgets, and run history
  first-class.
- Stay provider-neutral across Claude Code, Codex, Aider, Ollama, OpenCode, and
  future tools.
- Keep the non-profit/local-first trust story: no hosted keys, no hidden
  telemetry, no auto-push, no auto-merge.
- Build the best run inspector: timeline, logs, diffs, validation results,
  decisions, rework, and final report.

License note
MIT is fine for maximum simplicity. Apache-2.0 is stronger for this project if
you want explicit patent protection and better company contribution comfort.
If the project has few outside contributors today, switching from MIT to
Apache-2.0 or dual MIT OR Apache-2.0 is still relatively easy.
